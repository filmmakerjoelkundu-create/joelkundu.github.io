# Bug Fixes - May 12, 2026

## 🐛 Modal Positioning Issues Fixed

### Initial Problem: Modal Opening in Hero Section ✅ FIXED

**Problem:** When clicking the "Know More" button in the Selected Works section, the modal was opening up in the hero section instead of being centered on the viewport.

**Root Cause:** The JavaScript was trying to position the modal over the work section by calculating its position relative to the work section's bounding box, but the calculations were incorrect.

**Solution:** Removed the broken custom positioning logic. The modal now uses CSS positioning (`position: fixed` with flexbox centering) which centers it properly on the viewport.

---

### Updated Problem: Modal Should Align with Selected Works Section ✅ FIXED

**Problem:** After the initial fix, the modal was centered on the viewport, but the user wanted it positioned specifically relative to the Selected Works section header instead.

**Solution:** Updated the modal positioning to:
1. Get the bounding rectangle of the `.work` section
2. Account for scroll position with `window.pageYOffset`
3. Position the modal to cover the work section area:
   - `top`: Section top + scroll offset
   - `left`: Section left position
   - `width`: Section width
   - `height`: Section height
4. Reposition on window resize to maintain alignment

**Files Modified:**
- `script.js` - Modal open handler and resize handler

---

## 📝 Code Changes Summary

### Modal Open Handler:
```javascript
// Position modal relative to Selected Works section
const workSection = document.querySelector('.work');
if (workSection) {
  const rect = workSection.getBoundingClientRect();
  const scrollTop = window.pageYOffset || document.documentElement.scrollTop;
  
  // Position modal to cover the work section area
  stillsModal.style.position = 'fixed';
  stillsModal.style.top = `${rect.top + scrollTop}px`;
  stillsModal.style.left = `${rect.left}px`;
  stillsModal.style.width = `${rect.width}px`;
  stillsModal.style.height = `${rect.height}px`;
}

stillsModal.classList.add('active');
// No scroll lock - user can scroll while modal is open
```

### Resize Handler:
```javascript
window.addEventListener('resize', () => {
  if (stillsModal.classList.contains('active')) {
    const workSection = document.querySelector('.work');
    if (workSection) {
      const rect = workSection.getBoundingClientRect();
      const scrollTop = window.pageYOffset || document.documentElement.scrollTop;
      
      stillsModal.style.top = `${rect.top + scrollTop}px`;
      stillsModal.style.left = `${rect.left}px`;
      stillsModal.style.width = `${rect.width}px`;
      stillsModal.style.height = `${rect.height}px`;
    }
  }
});
```
